# 4.1 UI 개선 (팝오버 캘린더 등)

> 이미지 기반 피드백과 반복적 개선 프로세스를 통해 학습 트래커의 UI를 다듬습니다.

## 스크린샷/이미지 기반 작업

Claude Code는 이미지를 이해할 수 있습니다. 터미널에 스크린샷을 붙여넣으면 Claude가 현재 UI 상태를 파악하고, 구체적인 개선안을 제시할 수 있습니다.

### 효과적인 UI 요청 방법

UI 개선을 요청할 때는 막연한 지시보다 구체적인 설명이 훨씬 좋은 결과를 만듭니다.

| 비효과적인 요청 | 효과적인 요청 |
|---------------|-------------|
| "이쁘게 만들어줘" | "날짜 선택에 shadcn/ui의 Popover + Calendar를 사용해줘" |
| "대시보드 개선해줘" | "과목별 총 학습 시간을 색상 코드 막대 차트로 보여줘" |
| "더 좋게 만들어줘" | "레이아웃을 반응형으로 만들어서 모바일에서는 1열, 데스크탑에서는 2열로 보여줘" |

## Step 1: 대시보드 UI 개선

현재 대시보드 페이지에 날짜 필터, 통계 카드, 주간 요약 등을 추가합니다.

### Claude에게 요청

```
대시보드 페이지를 개선해줘:
- shadcn/ui Popover + Calendar을 사용해서 날짜별 세션 필터링을 위한 날짜 선택기 추가
- 과목별 총 학습 시간을 색상 코드 막대 차트로 보여주는 과목 통계 카드 추가
- 간단한 차트로 주간 학습 요약 섹션 추가 (div 바 사용, 차트 라이브러리 불필요)
- 레이아웃이 반응형인지 확인
```

### 예상 결과

Claude는 다음과 같은 작업을 수행합니다:

1. 필요한 shadcn/ui 컴포넌트 설치 (`npx shadcn@latest add popover calendar`)
2. 날짜 선택 Popover + Calendar 컴포넌트 추가
3. 과목별 통계 카드 구현 (색상 코드 막대 차트)
4. 주간 학습 요약 섹션 추가 (div 기반 bar chart)
5. 반응형 레이아웃 적용

{% hint style="info" %}
shadcn/ui의 Calendar 컴포넌트는 내부적으로 `react-day-picker`를 사용합니다. Claude가 자동으로 필요한 의존성을 설치할 것입니다.
{% endhint %}

## Step 2: 세션 생성 폼 개선

세션 생성 폼도 더 사용하기 편하게 개선합니다.

### Claude에게 요청

```
새 세션 생성 폼을 개선해줘:
- 일반 날짜 입력 대신 Popover + Calendar로 날짜 선택
- 시작/종료 시간을 위한 시간 선택기 추가
- 여러 과목과 학습 블록을 인라인으로 추가 가능하게
- 사용자가 블록을 채울 때 총 학습 시간 미리보기 표시
```

### 핵심 포인트

- **인라인 편집**: 과목과 학습 블록을 폼 안에서 바로 추가/삭제
- **실시간 합산**: 블록을 입력할 때마다 총 학습 시간 미리보기
- **UX 개선**: 일반 input 대신 캘린더와 시간 선택기 사용

## Step 3: 반복적 개선 프로세스

첫 번째 결과가 완벽하지 않을 수 있습니다. Claude Code를 활용한 UI 개선은 반복적인 프로세스입니다.

### 구체적으로 수정 요청하기

첫 결과가 마음에 들지 않으면, 구체적으로 수정을 요청합니다:

```
날짜 선택기가 입력 필드 위가 아니라 아래에 열려야 해.
그리고 캘린더에서 오늘 날짜와 현재 선택된 날짜가 하이라이트되어야 해.
```

```
막대 차트의 바가 너무 얇아. 최소 24px 너비로 만들고
각 바 위에 학습 시간 값(예: "2h 30m")을 라벨로 추가해줘.
```

```
통계 카드는 모바일에서 2열, 데스크탑에서 4열 그리드여야 해.
현재 모두 한 줄에 있어서 모바일에서 오버플로되고 있어.
```

### 2번 실패 시 /clear 후 재시작

같은 수정을 2번 이상 요청했는데도 원하는 결과가 나오지 않는다면:

1. `/clear` 명령으로 컨텍스트를 초기화합니다
2. 원하는 결과를 처음부터 구체적으로 설명합니다
3. 필요하면 참고할 파일 경로를 직접 지정합니다

```
/clear
```

```
app/dashboard/page.tsx를 봐줘. 날짜 선택 팝오버가 올바르게 위치하지 않아.
shadcn/ui의 Popover 컴포넌트에서 side="bottom"과 align="start"를 사용해야 해.
Popover 위치를 수정해줘.
```

{% hint style="warning" %}
컨텍스트 창이 커지면 Claude의 응답 품질이 떨어질 수 있습니다. UI를 반복적으로 수정할 때는 `/clear`를 적극 활용하세요.
{% endhint %}

## Step 4: 이미지 피드백

Claude Code 터미널에서 스크린샷을 붙여넣어 시각적 피드백을 줄 수 있습니다.

### 사용 방법

1. 브라우저에서 현재 UI 스크린샷을 찍습니다
2. Claude Code 터미널에 이미지를 붙여넣습니다
3. 수정 요청을 함께 입력합니다

```
[스크린샷 붙여넣기]

이 스크린샷을 보면 통계 카드의 막대 차트가 카드 밖으로 넘치고 있어.
차트가 카드 너비 안에 맞게 조정해줘. 그리고 과목 이름이 길면 ... 으로 잘리게 해줘.
```

### 이미지 피드백 팁

- 문제가 되는 부분을 가리키는 것보다 텍스트로 설명하는 것이 더 효과적
- 전체 페이지보다 문제 영역만 캡처하면 더 정확한 수정 가능
- "이 부분을 저 디자인처럼 만들어줘"보다 구체적인 CSS 속성을 언급하면 좋음

## 정리

이 섹션에서 배운 내용:

- **구체적인 요청**이 좋은 결과를 만든다 (컴포넌트 이름, 레이아웃 수, 크기 등)
- **반복적 개선**: 첫 결과 확인 후 구체적으로 수정 요청
- **이미지 피드백**: 스크린샷으로 현재 상태를 Claude에 전달
- **/clear 활용**: 2번 이상 실패하면 컨텍스트 초기화 후 재시작
- **shadcn/ui**: Popover, Calendar 등 컴포넌트를 활용한 UI 개선
